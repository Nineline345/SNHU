### Briefly summarize your client, Artemis Financial, and its software requirements. Who was the client? What issue did the company want you to address?
The client was Artemis Financial, a financial services company that required an improved software security to protect the sensitive data customer of their customers. The main issue addressed was identifying security vulnerabilities and implementing secure communication practices such as HTTPS, certificate management, and data integrity verification using SHA-256.

### What did you do well when you found your client’s software security vulnerabilities? Why is it important to code securely? What value does software security add to a company’s overall well-being?
I performed well in identifying security risks and applying mitigation techniques such as cryptographic hashing, TLS encryption, and dependency analysis. Secure coding is critical because it protects data integrity, prevents unauthorized access, and reduces the risk of breaches. Strong software security adds long-term value by maintaining customer trust, regulatory compliance, and operational stability.

### Which part of the vulnerability assessment was challenging or helpful to you?
The most challenging part was correctly configuring certificates and secure communication while keeping the program functional. A helpful aspect was learning how to systematically assess risks using both manual review and automated tools. I believe the core though process systematic assessments will be applicable to both Software Security and other industry practices as well.

### How did you increase layers of security? In the future, what would you use to assess vulnerabilities and decide which mitigation techniques to use?
I increased security by implementing HTTPS with TLS and using SHA-256 for checksum verification. Additionaly, I avoided deprecated protocols, and performed dependency checks. In the future, I would continue using tools such as OWASP Dependency-Check, static analysis scanners, and design reviews that prioritize and select the right mitigation strategies.

### How did you make certain the code and software application were functional and secure? After refactoring the code, how did you check to see whether you introduced new vulnerabilities?
After refactoring, I validated my program's functionality through execution testing and verified the security through dependency-check reports and secure browser connections. This helped ensure that no new vulnerabilities were introduced during remediation.
