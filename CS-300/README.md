### What was the problem you were solving in the projects for this course?
The problem I was trying to solve in these projects was designing and implementing an efficient data structures to store, search, and sort course information, then analyzing their run-time and memory usage. The final goal of these projects was to correctly load, organize, and print the ABCU Computer Science course data in alphanumeric order.

### How did you approach the problem? Consider why data structures are important to understand.
I approched the problem by evalutating multiple data structures, such as vectors, hash tables, and binary search trees, and selected those that best balanced performance and memory usage. Understanding data structures was critical because the choice directly affected run-time, memory usage, and the program's scalability.

### How did you overcome any roadblocks you encountered while going through the activities or project?
I overcame challenges by breaking problems into smaller steps, validating logic with incremental testing, and using debugging output to verify data flow. Reviewing algorithm behavior and carefully tracing execution helped resolve issues with sorting, searching, and data loading.

### How has your work on this project expanded your approach to designing software and developing programs?
This project reinforced the importance of planning data flow and selecting appropriate algorithms early in the design process. I now approach software design with greater emphasis on efficiency and scalability.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
I place more focus on clear function separation, consistent naming conventions, and modular design. This makes the code easier to read and maintain, especially when swapping or modifying data structures without rewriting large portions of the program.
